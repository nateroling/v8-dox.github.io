704 // ES6 Draft Rev3 10.2 specifies declarative environment records with mutable
705 // and immutable bindings that can be in two states: initialized and
706 // uninitialized. In ES5 only immutable bindings have these two states. When
707 // accessing a binding, it needs to be checked for initialization. However in
708 // the following cases the binding is initialized immediately after creation
709 // so the initialization check can always be skipped:
710 // 1. Var declared local variables.
711 // var foo;
712 // 2. A local variable introduced by a function declaration.
713 // function foo() {}
714 // 3. Parameters
715 // function x(foo) {}
716 // 4. Catch bound variables.
717 // try {} catch (foo) {}
718 // 6. Function variables of named function expressions.
719 // var x = function foo() {}
720 // 7. Implicit binding of 'this'.
721 // 8. Implicit binding of 'arguments' in functions.
722 //
723 // ES5 specified object environment records which are introduced by ES elements
724 // such as Program and WithStatement that associate identifier bindings with the
725 // properties of some object. In the specification only mutable bindings exist
726 // (which may be non-writable) and have no distinct initialization step. However
727 // V8 allows const declarations in global code with distinct creation and
728 // initialization steps which are represented by non-writable properties in the
729 // global object. As a result also these bindings need to be checked for
730 // initialization.
731 //
732 // The following enum specifies a flag that indicates if the binding needs a
733 // distinct initialization step (kNeedsInitialization) or if the binding is
734 // immediately initialized upon creation (kCreatedInitialized).
735 enum InitializationFlag {
736  kNeedsInitialization,
737  kCreatedInitialized
738 };
